# Shird

**Shird** is a powerful and flexible authentication solution designed for modern applications.

## What is Shird?

**Shird** is a lightweight and extensible authentication package that simplifies login flows and token management.
It integrates seamlessly with your existing stack, offering a perfect balance between simplicity and flexibility.

This documentation will guide you through setup, configuration, and advanced features to help you integrate **Shird** efficiently.

<!-- We use **Shird** across multiple production-grade applications to handle authentication quickly and securely. -->

## Key Features

**Shird** includes a wide range of built-in features for secure, modern authentication:

- **Easy Integration** — Simple setup and flexible API for quick adoption.
- **Client Flexibility** — Define your own HTTP client and pass it to the shared layer (e.g., `dio`, `http`, etc.).
- **Storage Flexibility** — Choose your preferred token storage method (`secure_storage`, `shared_preferences`, etc.).
- **Local Login** — Supports traditional email/password authentication workflows.
- **Predefined Providers** — Includes popular providers (Microsoft, etc.) that you can use, extend, or override.
- **Token Management** — Built-in utilities for token generation, storage, and verification.
- **Persistent Sessions** — Automatic token refresh and session management.
- **Extensibility** — Easily add custom authentication strategies or modify existing ones.

Get started with **Shird** and implement secure, scalable authentication in minutes.
